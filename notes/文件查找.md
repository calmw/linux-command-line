#### locate

- locate 按照路径名查找文件（根据文件名查找）
- locate 命令会查找路径名数据库，输出所有包含该路径字符串的匹配选项
- locate 如果查找条件比较复杂，可以将locate命令与grep等其他命令结合起来使用
- locate 命令已经存在了很长一段时间了，也有几种常见的变体。slocate命令和mlocate命令是现代Linux发行版中常见的两种，不过通常是通过符号链接locate命令访问的。不同版本的locate有一些重叠的选项。

> ###### 在使用locate命令时，新创建的文件可能不会出现在查找结果中，可以切换到超级用户，手动执行updatedb。因为locate命令的数据库是通过另一个updatedb创建的。它通常作为cron作业，由守护进程cron定时执行，大多数包含locate的系统每天执行一次。

#### find

- 在目录中查找文件
- find ~  该命令将列出主目录所有文件
- type,文件类型选项, 限制只查找目录: ``` find ~ -type d ```,find支持的常见文件类型：

| 文件类型 | 描述     |
|------|--------|
| b    | 块设备文件  |
| c    | 字符设备文件 |
| d    | 目录     |
| f    | 普通文件   |
| l    | 符号链接   |

- 匹配通配符模式
    - 匹配文件名或目录后缀为.JPG，类型为文件，大小大于1M ``` find ~ -type f -name "*.JPG" size +1M | wc -l ```
    - 其中size的 +1M表示大于1MB，如果是-1M则为小于1MB，1M为等于1MB

- find 选项众多，这里就不一一列举了，只写简单常用的

#### touch

- touch 常用于设置或更新文件访问（assess）、变更（change）、修改（modify）时间。但如果指定的文件并不存在，则创建同名空文件
- touch 如果文件存在，修改文件时间；如果文件不存在就创建文件。
- 创建一个包含100个子目录（每个目录包含26个空文件）的playground目录。``` touch playgroung/dir-{001..100}/file{A..Z} ```
